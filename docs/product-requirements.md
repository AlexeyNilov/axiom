# Product Requirements

## Functional Requirements

### Artwork

* Store artwork metadata
* Display artwork image
* Support local and remote images

---

### Observation

Users can create observations.

Observations are factual.

Example:

"Three circles dominate the composition."

Not:

"The artist wants us to feel..."

---

### Principle

Users derive reusable principles.

Example:

"Hierarchy emerges from contrast."

Each principle references one or more observations.

---

### Software Mapping

Users connect principles to software concepts.

Examples

* API Design
* Domain Modeling
* UI
* Event Systems
* Architecture
* Testing

Mappings always contain rationale.

---

### Experiment

Each mapping may generate experiments.

Experiment contains

* hypothesis
* task
* expected outcome
* actual outcome
* status

---

### Reflection

Users record lessons learned.

These become searchable knowledge.

---

## Non-functional Requirements

* Offline-first architecture where practical
* Fast local search
* Extensible graph model
* Complete traceability
* Full test coverage of domain logic
